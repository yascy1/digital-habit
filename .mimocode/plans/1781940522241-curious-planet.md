# Fix: `pnpm dev` Error — `tw-animate-css` CSS Import Resolution Failure

## Root Cause

Error yang sama muncul untuk **SEMUA halaman** (dashboard, input-aktivitas, dll) karena error-nya ada di `app/globals.css:2`:

```
CssSyntaxError: Can't resolve 'tw-animate-css' in 'C:\Users\skesu\sib4a-web\digital-habit\app'
```

`globals.css` diimport oleh `app/layout.tsx` (root layout) — yang dipakai oleh semua halaman. Jadi error ini bukan per-branch, tapi error CSS global yang mempengaruhi seluruh app.

**Kenapa bisa terjadi:**
- Package `tw-animate-css` sudah ter-install di `node_modules`
- Tapi package-nya **hanya export CSS via `"style"` condition** di `exports` field (bukan `"default"`)
- PostCSS/Tailwind resolver tidak bisa resolve bare import `@import "tw-animate-css"` karena `exports` field tidak punya entry `"."` dengan `"default"` condition
- Harusnya import-nya pakai path eksplisit ke file CSS

## Fix

**File:** `app/globals.css:2`

Ubah:
```css
@import "tw-animate-css";
```

Menjadi:
```css
@import "tw-animate-css/dist/tw-animate.css";
```

Ini langsung menunjuk ke file CSS yang ada di package, bypassing broken exports resolution.

## Verification

1. Kill dev server lama: `taskkill /PID 7056 /F`
2. Jalankan `pnpm dev`
3. Buka `http://localhost:3000/dashboard` — harusnya load tanpa error
4. Buka `http://localhost:3000/input-aktivitas` — harusnya load tanpa error
5. Cek browser console — tidak ada CSS error lagi
