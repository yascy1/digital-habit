# Fix Hydration Error di AuthCheck

## Branch: `fitur-dashboard`

## File: `components/auth-check.tsx`

## Problem
`AuthCheck` return `null` di server (karena `useMemo`/`useState` awal = false), tapi return `children` di client setelah `useEffect` baca localStorage. Server HTML ≠ client HTML → hydration error.

## Fix
Render `children` SELALU (server DAN client). Redirect ke `/login` di `useEffect` saja jika tidak authorized.

```tsx
"use client"

import { useEffect, useState } from "react"
import { useRouter } from "next/navigation"

export function AuthCheck({ children }: { children: React.ReactNode }) {
  const router = useRouter()
  const [authorized, setAuthorized] = useState(true) // default true agar server = client

  useEffect(() => {
    const isAuth = !!localStorage.getItem("digital-habit-user")
    setAuthorized(isAuth)
    if (!isAuth) {
      router.replace("/login")
    }
  }, [router])

  return <>{children}</>
}
```

Server render: `<>{children}</>`
Client render: `<>{children}</>`
→ Sama. Tidak ada hydration error.

Redirect ke `/login` terjadi di `useEffect` (client-only). Ada brief flash layout sebelum redirect, tapi tidak ada error.

## Verification
- `npm run build` tanpa error
- Buka `/dashboard` tanpa login → redirect ke `/login` tanpa hydration error
- Login → dashboard tampil normal tanpa error
