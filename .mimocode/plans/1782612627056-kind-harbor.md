# Plan: Harden lupa-password against browser cross-fill

## Context
Previous turn added `autoComplete` attributes, but Chrome/Edge still cross-fills due to DOM reuse between steps. Need 5 additional anti-autofill techniques.

## File to Modify
- `app/lupa-password/page.tsx`

## Changes

### 1. Add `key={step}` on all three `<form>` elements
- Line 116: `<form ... key={step}>` (email)
- Line 153: `<form ... key={step}>` (verify)
- Line 194: `<form ... key={step}>` (reset)

This forces React to unmount/remount the entire form DOM subtree on step change.

### 2. `autoComplete="off"` on forms — ALREADY DONE
Lines 116, 153, 194 already have `autoComplete="off"`. No change needed.

### 3. Add `name="verify-name-field"` on name input (line 163-170)
Change from generic `name` (implied by register) to explicit non-generic:
```tsx
<Input
  id="name"
  type="text"
  name="verify-name-field"
  autoComplete="nope"
  ...
/>
```

### 4. Add `name` on password inputs (lines 204-211, 219-226)
- Password Baru: add `name="reset-password-field"`
- Konfirmasi Password: add `name="reset-confirm-password-field"`

### 5. Add hidden dummy input at top of each form
After the opening `<form>` tag, before the first visible `<div>`, add:
```tsx
<input type="text" className="hidden" tabIndex={-1} autoComplete="off" aria-hidden="true" />
```

Three locations:
- Email form (after line 116)
- Verify form (after line 153)
- Reset form (after line 194)

## What stays the same
- Zod schemas, onSubmit functions, step flow, all UI structure
