# Plan: Branch Management — fitur-input

## Context
User wants the input-aktivitas feature on a separate git branch called `fitur-input`, not mixed with the dashboard branch (`fitur-dashboard`).

## Current State
- Current branch: `fitur-dashboard`
- Uncommitted input-aktivitas files: `app/input-aktivitas/`, `components/ui/input.tsx`, `components/ui/textarea.tsx`, `components/ui/select.tsx`
- Dashboard files also modified: `app/dashboard/page.tsx`, `app/globals.css`, `components/app-sidebar.tsx`

## Steps

### 1. Commit dashboard changes on `fitur-dashboard`
Commit the current dashboard modifications so they're saved before branching:
- `app/dashboard/page.tsx`
- `app/globals.css`
- `components/app-sidebar.tsx`
- `components/ui/select.tsx`

### 2. Create `fitur-input` branch from `fitur-dashboard`
```
git checkout -b fitur-input
```

### 3. Commit input-aktivitas files on `fitur-input`
- `app/input-aktivitas/layout.tsx`
- `app/input-aktivitas/page.tsx`
- `components/ui/input.tsx`
- `components/ui/textarea.tsx`

### Result
- `fitur-dashboard` branch: has dashboard changes only
- `fitur-input` branch: has dashboard changes + input-aktivitas feature

## Verification
- `git branch` shows both branches
- `git log --oneline fitur-dashboard` shows dashboard commits
- `git log --oneline fitur-input` shows input-aktivitas commit on top
- `pnpm dev` works on both branches
